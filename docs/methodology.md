# 数据方法论

## 1. 分类原则

知识库使用九大板块进行研究分类，但产业链关系由有向边表达，不假设所有节点处于单一直线层级。一家公司可关联多个产业节点，一个节点也可以同时服务多个下游。

## 2. 节点口径

产业节点必须是可独立描述的产品、技术、设备、材料、软件或服务，并至少说明：

- 解决的问题和边界；
- 核心技术；
- 代表产品或直接产出物；
- 直接上游投入和直接下游用途；
- 供应集中度、技术壁垒或替代难度。

## 3. 上市公司口径

覆盖 A 股、港股、美股、台股、日股、韩股及欧洲主要交易所。公司必须直接提供相关产品、技术或服务并形成商业收入。以下情况排除：

- 仅使用 AI 提升内部效率；
- 只有概念性规划、未形成产品或收入；
- 仅通过少量参股间接相关；
- 非上市公司。

目录是截至核验日期的研究快照，不代表交易所实时证券名录。

## 4. 来源优先级

1. 监管机构、交易所申报、招股书和年度报告；
2. 公司投资者关系资料和官方产品页面；
3. 官方标准组织、政府机构和行业协会；
4. 其他来源只能作为检索线索，不能单独支撑关键结论。

## 5. 财务数据

- 保留原财年、原币种和披露口径；
- 营收、毛利率和净利润必须能够追溯到正式披露；
- AI 相关收入占比只记录公司明确披露的数值；
- 市值必须同时记录估值日期；
- 没有披露或尚未逐项核验时显示“未披露”，不推算、不补齐。

## 6. 更新与质量检查

每次更新应运行：

```powershell
node tests/catalog.test.mjs
node tests/dashboard-structure.test.mjs
node tests/app-contract.test.mjs
```

检查内容包括重复 ID、悬空关系、缺失证券代码、无产业节点关联的公司、非上市公司混入和页面核心契约。
